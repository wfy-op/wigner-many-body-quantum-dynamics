# 项目工作流记忆

## GitHub 发布约定

- 每次完成一个经过验证、可交付的全书修订版本后，默认将相关提交推送到本仓库的 `main` 分支并创建新的 GitHub Release；用户另有说明时除外。
- Release 标签采用 `v<主版本>.<次版本>.<修订号>-YYYY-MM-DD`，重大内容扩充至少递增次版本号。
- Release 必须附带编译好的 PDF，附件文件名使用“书名_YYYY-MM-DD.pdf”。
- Release 说明保持简洁，并明确列出相对上一个 Release 的主要变化、验证结果和仍保留的科学边界。
- 发布前确认工作区范围、构建成功、结构化数值门禁、PDF 页数与 SHA-256；发布后读取远端 Release 和附件信息复核。
